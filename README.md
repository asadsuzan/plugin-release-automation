# 🚀 Plugin Release Automation Bot

A powerful, high-performance automation suite for WordPress developers to manage releases, generate changelogs, and visualize release history across multiple plugins with a single source of truth.

---

## 🏛️ System Architecture

The system operates on a **Single Source of Truth** model. All release data is authored in structured Markdown files, which are then processed to generate various distribution formats.

- **Source of Truth**: `plugins/{slug}/releases/{version}.md`
- **Processing Engine**: TypeScript & Node.js
- **Frontend Layer**: React + Vite (Modern Dark Dashboard)
- **Data Sync**: Local JSON caches for fast dashboard performance

### 🛠️ Technology Stack
- **Runtime**: Node.js with `ts-node`
- **Parsing**: `gray-matter` (YAML Frontmatter), `cheerio` (HTML Scraping)
- **CLI**: `inquirer` (Interactive Workflows), `chalk` (Terminal Styling)
- **Dashboard**: React 18, Vite, Lucide Icons, Tailwind CSS (for dashboard styling)

---

## 🚀 Key Features

### 1. 📦 Bulk WordPress.org Import
Automatically scaffold your entire plugin portfolio in seconds.
- **Profile Scraping**: Enter a WP.org username and fetch all public plugins.
- **Historical Backfilling**: Automatically pulls `readme.txt` from the SVN trunk and converts the entire changelog history into structured `.md` files.
- **Smart Date Interpolation**: Missing dates in legacy changelogs are intelligently interpolated to maintain a realistic historical timeline.

### 2. ⚡ Automated Release Workflow (`npm run release`)
One command to rule them all. When you publish a release:
- **Patch Readme**: Updates the local `readme.txt` with the latest changes using configurable formats.
- **Sync Changelog**: Appends the release to `generated/changelog.json` for in-plugin admin dashboards.
- **Generate Reports**: Aggregates releases into monthly reports (Markdown & JSON) for stakeholders.

### 3. 🖥️ Visualization Dashboard
A premium, dark-mode dashboard built with a modern React architecture to visualize your release velocity and metadata.
- **Architecture**: Modular React 18 with `react-router-dom` and a specialized data service layer.
- **Styling**: Tailwind CSS for a high-performance, glassmorphism-inspired UI.
- **Insights**: Track release types (Major, Minor, Patch, Hotfix) and tag distribution with interactive timelines.
- **Real-time Sync**: Automatically fetches data from the `public/data/` registry generated by the engine.

#### Dashboard Commands
1. Run `npm run dashboard` (from root) to sync data and start the dev server.
2. Visit `http://localhost:5173` to explore:
    - **Overview**: System health and release velocity.
    - **Plugins**: Detailed registry management.
    - **Releases**: Searchable, interactive timeline with expanded release notes.

### 4. 📝 Configurable Output
Control every aspect of the output via `config.json`:
- Custom date formats (e.g., `d-m-Y`).
- Bullet point styles (`*` vs `-`).
- Spacing and indentation rules.

## 💻 Available Commands

| Command | Description |
| :--- | :--- |
| `npm run new-release` | Start the interactive wizard to import plugins or scaffold a new release. |
| `npm run release` | Process all "published" releases and update assets. |
| `npm run release:dry` | Preview release changes without writing to files. |

## 🔧 Technical Implementation Notes

### Release Parsing Logic
The parser uses a hybrid approach:
1. **Frontmatter**: Extracts metadata like version, date, type, and tags.
2. **Markdown Sections**: Parses `# Features`, `# Improvements`, and `# Fixes` headers.
3. **Item Parsing**: Supports both simple list items and structured key-value pairs (title, category, screenshot, etc.).

### WP.org Sync
To ensure maximum compatibility, the sync script:
- Emulates a modern browser User-Agent.
- Includes a 24-hour cache for profile scraping to avoid rate limits.
- Uses recursive regex to isolate changelog blocks from SVN `readme.txt` files.
